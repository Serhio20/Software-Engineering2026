1. Insurance Policy Workflow
Description: Activity diagram illustrating the insurance purchase process.
Key Components:

Initialization: User profile validation (new vs. existing).

Selection: Coverage options and discount application.

Payment: Payment gateway integration with error handling.

Completion: Policy generation, automated notification (Email/SMS), and database update.

2. ATM Transaction Sequence
Description: Sequence diagram detailing the interaction flow during an ATM withdrawal.
Key Components:

Validation: Card insertion and PIN authentication.

Processing: Transaction selection, bank server communication, and balance check.

Execution: Account debit and transaction success verification.

Termination: Receipt printing and card ejection.

3. Hospital Patient Workflow
Description: Activity diagram modeling the patient lifecycle within a medical facility.
Key Components:

Admission: Registration, insurance eligibility, and appointment scheduling.

Consultation: Triage, vitals assessment, and doctor examination.

Clinical Path: Diagnostic testing (if required) and medication prescription.

Discharge: Invoice generation, payment handling, and EMR (Electronic Medical Record) update.
